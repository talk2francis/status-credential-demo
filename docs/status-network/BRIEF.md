# Status Network Build Brief

## Goal
Design and ship an AI-assisted experience that plugs into Status Network primitives to help non-technical professionals earn public crypto status via guided learning, sandbox practice, and verifiable attestations.

## Hypothesis
- Professionals outside core crypto circles need a low-friction way to prove progress.
- Status Network (SNT) + attestations can provide portable reputation if fed with trusted learning signals.
- Our agent + Status Credential Demo sandbox can generate those signals automatically (less manual proof collection).

## Users
1. **Learners** – pharmacists, students, traders entering crypto.
2. **Instructors / status curators** – guide learning paths, validate projects.
3. **Ecosystem partners** – protocols or DAOs needing credentialed contributors.

## Must-haves (aligned with Synthesis deliverables)
- Guided modules tied to Status Network incentives.
- Sandbox-linked proofs → mintable attestations (Base / Status).
- Social sharing artifacts optimized for Status messenger + wider networks.
- Admin/analytics views for campaign health.

## Unknowns / Research Tasks
- Status Network APIs / SDKs available for hackathon use.
- Recommended contract stack (SNT, ENS usernames, Keycard, etc.).
- Specific bounty requirements (tracks, judging metrics).

## Next Actions
1. Collect latest Status Network docs + bounty guidelines.
2. Map Status Credential Demo flows → Status credentials (what equals “status”? XP thresholds, attestations, NFTs?).
3. Draft technical architecture (frontend, attestations, optional contracts).
4. Break build into milestones with 14-day cadence.
5. Identify any dependencies (RPC, testnets, design assets).
